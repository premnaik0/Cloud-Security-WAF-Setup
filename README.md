# AWS WAF Sandbox: Web Application Firewall Lab

AWS WAF is a web application firewall that helps protect web applications or APIs against common web exploits and bots that may affect availability, compromise security, or consume excessive resources.

In this lab, I built a custom Web ACL named "learning-web-acl" targeting regional infrastructure in the Ohio (us-east-2) region.

### Key Learning Objectives

- Provisioning a Web ACL from scratch using custom, cost-optimized rule configurations.
- Writing and inspecting a custom header-matching rule logic.
- Navigating the JSON rule editor to understand Infrastructure-as-Code (IaC) signatures.
- Exploring the AI and Bot Traffic Analysis dashboard to monitor automated scrapers.

---

## Configuration & Implementation Steps

### 1. Creating the Web ACL Pack

Using the AWS WAF console, I created a regional security pack designed for standard web applications. I selected "Build your own pack from all of the protections AWS WAF offers" to construct a lean, customized security layer without automatically enrolling in paid managed rule groups.

- **Web ACL Name:** "learning-web-acl"
- **Resource Scope:** Regional (Ohio / us-east-2)
- **Associated App Focus:** Both API and Web traffic

---

### 2. Custom Rule Implementation: "block-test-header"

I authored a custom matching rule to inspect incoming HTTP request headers. If a request carries a header matching specific testing parameters, the WAF immediately intercepts and blocks it with an HTTP 403 Forbidden response.

#### Custom Rule Rule Logic (JSON)

The rule evaluates traffic using a strict byte-match evaluation schema. Below is the exact JSON structure generated by the AWS WAF engine for this rule:

```json
{
  "Name": "block-test-header",
  "Priority": 0,
  "Statement": {
    "ByteMatchStatement": {
      "SearchString": "block-me",
      "FieldToMatch": {
        "Headers": {
          "MatchPattern": {
            "All": {}
          },
          "MatchScope": "ALL",
          "OversizeHandling": "CONTINUE"
        }
      },
      "TextTransformations": [
        {
          "Priority": 0,
          "Type": "NONE"
        }
      ],
      "PositionalConstraint": "EXACTLY"
    }
  },
  "Action": {
    "Block": {}
  },
  "VisibilityConfig": {
    "SampledRequestsEnabled": true,
    "CloudWatchMetricsEnabled": true,
    "MetricName": "block-test-header"
  }
}
```

### 3. AI Traffic & Bot Management Analysis

AWS WAF features built-in telemetry to monitor automated traffic. I explored the AI Traffic Monetization and Analysis dashboard, which acts as a specialized pane to track:

- **Verified AI bot traffic** (e.g., search engine crawlers, official indexers).
- **Unverified AI bot traffic** (e.g., scrapers violating robots.txt).
- **Top crawlers by name** and their request volume, enabling fine-grained control over which automated agents can scan application assets.
